## Decision Framework (Day 30)

**Ask yourself these 3 questions:**

1. **Does the app work?**
   - Zero critical bugs blocking core flows?
   - YES → Continue | NO → Emergency fixes

2. **Are people using it?**
   - >1,500 installs despite minimal marketing?
   - YES → Scale marketing | NO → Hold steady

3. **Are people paying?**
   - >10 agent subscriptions or first bookings?
   - YES → Revenue model works | NO → Refine pricing

**If all 3 YES → GO (full marketing spend, scale team)**
**If 2/3 YES → HOLD (fix issues, relaunch week 6)**
**If <2 YES → PIVOT (reevaluate or sunset)**

## Glossary

- **DAU** = Daily Active Users
- **MRR** = Monthly Recurring Revenue
- **MVP** = Minimum Viable Product
- **CAC** = Customer Acquisition Cost
- **D7 Retention** = % of users still active 7 days after install
- **NPS** = Net Promoter Score (how likely to recommend)
- **UTM** = Tracking parameters (utm_source, utm_campaign)
- **FCM** = Firebase Cloud Messaging (push notifications)
- **APNs** = Apple Push Notification service
- **WebSocket** = Real-time bidirectional communication
- **PostGIS** = PostgreSQL extension for geographic queries
- **SINPE Móvil** = Costa Rica's national mobile payment system
