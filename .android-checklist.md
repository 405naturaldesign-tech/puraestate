# Android Build & Play Store Deployment Checklist

## Pre-Submission Configuration Verification

### Required Configuration Files

- [ ] **eas.json**
  - [ ] Production profile uses `"buildType": "aab"` for Google Play Store
  - [ ] Gradle command set to `:app:bundleRelease` for production
  - [ ] Service account credentials configured for Android submission
  - [ ] Release track set to "production" for production builds

- [ ] **app.json**
  - [ ] Version number: "1.0.0" (or appropriate version)
  - [ ] App name: "PuraEstate"
  - [ ] Slug: "pura-estate"
  - [ ] Android package: "com.pura.estate"
  - [ ] All required permissions listed (CAMERA, LOCATION, STORAGE, INTERNET, etc.)
  - [ ] Adaptive icon properly configured with foregroundImage and backgroundColor
  - [ ] Intent filters configured for deep linking
  - [ ] Google Services file reference: "./google-services.json"

- [ ] **Android Native Configuration**
  - [ ] AndroidManifest.xml properly configured
  - [ ] google-services.json present in root directory
  - [ ] Signing certificate configured in local environment
  - [ ] .netrc file present for Gradle dependencies (if applicable)

### Play Store Listing Assets (Required Graphics)

#### Mandatory Assets
- [ ] **App Icon** (512x512 PNG)
  - Location: `./assets/icon.png`
  - Format: PNG with transparency
  - All corners must be completely transparent

- [ ] **Feature Graphic** (1024x500 PNG)
  - Used in Play Store listing header
  - High-quality promotional image
  - Include app name and key features

- [ ] **Screenshots** (Minimum 2, maximum 8)
  - Dimensions: 1080x1920 pixels (or 1440x2560)
  - Formats: PNG or JPG
  - Required languages: English (US minimum)
  - Include 2-4 key app features per screenshot
  - Consider phone/tablet variations

#### Additional Assets
- [ ] **Promo Graphic** (180x120 PNG)
  - For Google Play Store promotional use

- [ ] **Video Preview** (MP4)
  - Optional but recommended
  - 15-30 seconds
  - Shows app functionality

### Play Store Listing Information

- [ ] **App Title**
  - Maximum 50 characters
  - Current: "PuraEstate"

- [ ] **Short Description**
  - Maximum 80 characters
  - Example: "Premium real estate platform for property discovery"

- [ ] **Full Description**
  - Maximum 4,000 characters
  - Include key features, benefits, and unique selling points
  - Mention permissions usage (camera, location, photos)

- [ ] **Category**
  - Select: "Shopping" or "Real Estate" (if available)

- [ ] **Content Rating Questionnaire**
  - Complete all required questions
  - Verify no content violations

### Android Permissions Verification

Verify these permissions are declared and justified:
- [ ] CAMERA - For property photo capture
- [ ] ACCESS_FINE_LOCATION - For nearby property search
- [ ] ACCESS_COARSE_LOCATION - For location-based features
- [ ] READ_EXTERNAL_STORAGE - For photo import
- [ ] WRITE_EXTERNAL_STORAGE - For photo export
- [ ] INTERNET - For API connectivity
- [ ] ACCESS_NETWORK_STATE - For network status checks
- [ ] VIBRATE - For haptic feedback
- [ ] RECORD_AUDIO - For video calls with agents
- [ ] MODIFY_AUDIO_SETTINGS - For call audio management
- [ ] USE_FULL_SCREEN_INTENT - For notification display
- [ ] SCHEDULE_EXACT_ALARM - For appointment reminders

### Build Validation Checklist

- [ ] Clean build generates `.aab` file (not `.apk`) for production
- [ ] Bundle size is under Google Play Store limits (150MB uncompressed)
- [ ] All assets properly embedded and referenced
- [ ] No hardcoded credentials in build artifacts
- [ ] Gradle build succeeds without warnings
- [ ] Version code incremented from previous release
- [ ] Signing certificate fingerprints verified

- [ ] Internal testing build deployed to Play Console
- [ ] Functionality tested on various Android versions (API 24+)
- [ ] Permissions tested and functioning correctly
- [ ] Deep linking tested with intent filters
- [ ] Firebase services configured and tested
- [ ] Photo upload/capture functionality verified

- [ ] Video call capabilities tested (if applicable)

### Production Release Checklist

- [ ] Privacy Policy URL provided and complies with Google Play policies
- [ ] Terms of Service URL provided

- [ ] Target API level meets Google Play requirements (currently API 24+)
- [ ] All graphics assets uploaded and reviewed
- [ ] Store listing text reviewed for clarity and grammar
- [ ] Release notes prepared for version update

### Post-Release Monitoring

- [ ] Monitor crash reports in Play Console
- [ ] Review user ratings and feedback
