COMMITLINT CONFIGURATION (.commitlintrc.json)
─────────────────────────────────────────────────────────────────────────────

Valid commit types:
  • feat       (New feature)
  • fix        (Bug fix)
  • docs       (Documentation)
  • test       (Tests)
  • refactor   (Code refactoring)
  • style      (Code style - no functional change)
  • chore      (Dependencies/build)
  • ci         (CI/CD changes)
  • perf       (Performance improvements)
  • revert     (Revert previous commit)

Message rules:
  • Header max length: 72 characters
  • Body max line length: 100 characters
  • Subject case: lowercase
  • Type required: always
  • Subject required: always
  • Subject full stop: never allowed
  • Body blank line: required if body present

LINT-STAGED CONFIGURATION (in package.json)
─────────────────────────────────────────────────────────────────────────────

Files patterns and actions:
  • *.{ts,tsx}  → eslint --fix, prettier --write
  • *.{js,jsx}  → eslint --fix, prettier --write
  • *.json      → prettier --write
  • *.md        → prettier --write

         $ chmod +x .husky/*

Solution: $ npm install
         $ npx husky install

Problem: Different errors locally vs hook
Solution: $ rm -rf node_modules package-lock.json
         $ npm install

Problem: Hook files missing
Solution: Check .gitignore doesn't exclude .husky/
         $ git add .husky/
         $ git commit -m "chore: add git hooks"

UPDATING HOOKS
─────────────────────────────────────────────────────────────────────────────

When .husky files are changed in repo:
  1. Git will pull latest versions
  2. No manual update needed
  3. Hooks automatically use new rules

When npm packages update:
  1. $ npm install          # Gets new versions
  2. $ npx husky install    # Ensures hooks use new packages
  3. Hooks automatically reflect new rules

ADDING NEW RULES
─────────────────────────────────────────────────────────────────────────────

To add new ESLint rule:
  1. Update .eslintrc.json
  2. Test locally: $ npm run lint
  3. Commit changes
  4. Team members auto-get update on next npm install

To add new commit type:
  1. Update .commitlintrc.json
  2. Test: make a commit with new type
  3. Commit changes
